# electron demo

## 学习路线

### 第一步：克隆 ElecDeep
- 跑起来项目，理解 Electron 主进程与渲染进程的通信机制
- 重点掌握 IPC 通信方式：`ipcMain` / `ipcRenderer`
- 研究如何在 Electron 中实现流式 SSE（Server-Sent Events）
- 建议补一个小 demo：主进程发送流式数据，渲染进程实时展示

### 第二步：克隆 Pointer
- 学习 React + TypeScript 在 Electron 里的工程化实践
- 关注构建方案、热重载、页面路由及状态管理
- 研究多模型切换的架构设计与实现方式
- 观察模型配置、请求分发和切换流程

### 第三步：克隆 Proma
- 学 AI Agent 系统如何在桌面端落地
- 重点看 MCP / Skills 的可扩展架构设计
- 理解技能注册、调用与扩展方式
- 学习本地数据持久化方案与历史数据管理

### 第四步：自己写一个
- 结合 DeepSeek-TUI 的灵感，做一个自己的 AI 桌面工具
- 先做最小可用版本（MVP）：基础聊天/任务、模型切换、历史记录
- 保持主进程与渲染进程职责清晰，避免逻辑混淆
- 逐步扩展插件式技能、更多模型、数据持久化

## 优化建议
1. 先理解 Electron 核心，再看工程化和 AI 架构。
2. 用小 Demo 验证每一项核心技术：IPC、SSE、React+TS、模型切换、Agent/Skills。
3. 把每个仓库的核心点做成笔记，便于后续整合到自己项目中。
4. 最后做自己的项目时，用迭代方式完成：先能跑，再优化架构，再补功能。
